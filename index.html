Arquitetura e Padrões de Projeto

O framework foi estruturado focando em escalabilidade, fácil manutenção e execução rápida em esteiras de CI/CD. Clique nos cards para explorar os conceitos.

Page Object Model (POM)

Padrão de design para estruturação de testes de UI.

Separação clara entre a lógica de negócio (ações na página) e os scripts de teste. Facilita enormemente a manutenção caso mapeamentos de tela ou seletores mudem, evitando refatoração em massa.
Clique para expandir

Data-Driven Testing (DDT)

Injeção dinâmica de massas de dados.

Uso de arquivos fixtures (formato JSON) para separar os dados de teste do código. Permite rodar o mesmo cenário de teste múltiplas vezes com diferentes conjuntos de dados de forma ágil.
Clique para expandir

Shift-Left Testing

Antecipação da qualidade no ciclo de vida.

Testes de API isolados rodando antes dos testes de UI para feedback rápido (Fail Fast). Identifica problemas na lógica de backend e integrações antes mesmo da interface gráfica estar pronta.
Clique para expandir

Custom Commands

Encapsulamento de rotinas complexas.

Criação de comandos customizados para ações repetitivas (como Login sistêmico, Geração de Token Auth, Setup de Massa). Mantém os arquivos de teste (specs) limpos e focados no comportamento.
Clique para expandir

Stack Tecnológica

Ferramentas e linguagens integradas para garantir uma cobertura abrangente, desde a interface do usuário até os contratos de dados em background.

Web/UI Automation

Cypress, Playwright (JS/TS), Allure/MochaAwesome para reports.

API Automation

REST Assured, Postman (Newman). Validação de JSON Schema e Status Code.

CI/CD Pipeline

GitHub Actions rodando testes automaticamente via PR ou Push na branch principal.

Workspace & Execução

Explore como estruturamos o projeto e os comandos essenciais para execução em ambiente local ou CI.

# 1. Clone o repositório
➜ ~ git clone https://github.com/seu-usuario/qa-automation-framework.git
# 2. Instale as dependências
➜ ~ npm install
# 3. Execução Headless (Background / CI)
➜ ~ npx cypress run
> Running: test-suite.cy.js (1 of 1)
> ✓ Login API validates token (120ms)
> ✓ UI renders dashboard correctly (850ms)
> All specs passed!
# 4. Execução Visual (Interface de Testes)
➜ ~ npx cypress open
