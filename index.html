Arquitetura e Padrões de Projeto

O framework foi estruturado com foco em escalabilidade, fácil manutenção e execução rápida em esteiras de CI/CD. Clique nos cards abaixo para explorar os conceitos arquiteturais aplicados neste repositório.

Page Object Model (POM)

Adoção do padrão POM para garantir uma separação clara entre a lógica de negócio (ações na página) e os scripts de teste. Isso facilita a manutenção: se um seletor mudar na interface da aplicação, a alteração é feita em um único arquivo, sem quebrar os cenários de teste associados.

Data-Driven Testing (DDT)

Uso de arquivos fixtures no formato JSON para injeção de massa de dados dinâmicos. Essa abordagem permite rodar o mesmo cenário de teste múltiplas vezes com conjuntos de dados diferentes, validando comportamentos de borda e regras financeiras complexas sem duplicar código.

Shift-Left Testing

Aplicação da cultura de prevenção de defeitos. O framework realiza testes de API isolados a correr antes dos testes de UI para feedback rápido. Validar os contratos de backend primeiro garante que falhas estruturais sejam capturadas antes do carregamento visual, economizando tempo de execução em CI.

Custom Commands

Criação de comandos customizados para abstrair ações repetitivas e complexas (ex: setup de ambiente, geração de tokens JWT, login de sistema). Isso mantém os arquivos de especificação (specs) limpos, semânticos e focados puramente na validação do comportamento do usuário.
