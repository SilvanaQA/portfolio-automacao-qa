📌 Sobre o Projeto
Este repositório é um portfólio de demonstração das minhas competências em Engenharia de Qualidade e Automação de Testes.
Devido a rigorosos acordos de confidencialidade (NDA) durante os meus 6 anos de atuação no setor bancário (Bradesco), os códigos reais de produção não podem ser expostos. Por conseguinte, este framework foi construído de raiz para testar aplicações públicas, demonstrando as melhores práticas de mercado que utilizo no meu dia a dia para garantir a resiliência de sistemas críticos.

🏗️ Arquitetura e Padrões de Projeto
O framework foi estruturado com um foco claro na escalabilidade, facilidade de manutenção e execução rápida em esteiras de integração contínua (CI/CD):
Page Object Model (POM): Separação clara entre a lógica de negócio (ações na página) e os scripts de teste.
Data-Driven Testing (DDT): Utilização de ficheiros fixtures (JSON) para a injeção dinâmica de massa de dados.
Shift-Left Testing: Testes de API isolados a executar antes dos testes de UI, garantindo um feedback rápido (prevenção de defeitos).
Custom Commands: Criação de comandos personalizados para abstrair ações repetitivas e complexas.

🛠️ Stack Técnica
Automação UI: Cypress, Playwright
Testes de API: REST Assured, Postman, Validações JSON
Base & Dependências: Node.js, JavaScript/TypeScript
Integração Contínua: GitHub Actions para execução automatizada
